Stage 2.3: Input -> Function -> Output

How do functions take input and produce output and what role do the keywords def and return have to do with this process?

  • it appears to me "def" defines the input or how to process the input
  • return produces an output

How to use a Procedure

  • Inputs can be called operands or arguements.
  • <procedure> (<input>, <input>, ...) is an example of how procedures are formatted.
  • The following are examples: 
 
            find ('au', 3)
            def rest_of_string (s):
              return s[1:]
            print rest_of_string ('audacity')
          
            udacity
  • Returns tell the defined procedure what to do with the inputs. Essentially it produces the output.

Types of Data to Input into a Function

  • When we use parentheses in a function there is no quotes "". That is because we are defining the function.
  • When we use parentheses when using a function (like the print command) we use quotes "" to define the input.

Making vs. Using Functions

  • When we make a function we are only telling it what it should do with the parameters it will receive when they are called. The parameter is just a placeholder.
  • When we use the function the real input values are placed into the where the parameter is inside the function.
  • In the procedure listed above, the string ‘audacity’ was the real value and s was the parameter.

Avoid Repetition

Functions help us to define a procedure that will work for multiple inputs therefore saving us lots of time for things that need the same formatting or conditions.
