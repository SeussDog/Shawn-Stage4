4.1 Introduction to Networks

Definition

A network is a group of entities that can communicate, even though they are not all directly connected.

Flooding

Flooding the network is where all nodes get the same message.

We need the following for a Network:

  1. ways to encode and interpret messages
    • Internet: message —> bits —> electrons/photons
  2. route messages
    • Internet: routers figure out next hops
  3. rules for deciding who gets to use resources
    • Internet: best effort service

Measuring Networks:

  1. Latency
    • The time it takes messages to get from source to destination: Unit of time such as milliseconds
  2. Bandwidth
    • amount of information that can be transmitted per unit of time: Units bits per second such as Megabit per second.

Bits

Bit is one unit of information in the form of a 0 or 1, like a yes/no question.

Protocols

  1. A method for communicating between a client and a server
  2. Hypertext Transfer Protocol (HTTP)
    • Example: client sends a ‘get' message to the server, finds file and sends it back to the client.
