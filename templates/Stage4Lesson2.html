4.2 Make the Internet Work for You

URL: Uniform Resource Locator

Example = http://www.udacity.com/cs253s/hipmunle.png
  • http = the protocol
  • www.udacity.com = the host
  • /cs253s/hipmunle.png = path to file

Query Parameter (GET parameters)

Example = http://example.com/foo?P=1&q=neat
  • Name = value: which is everything after the question mark

Fragment: something separated by the URL by a # sign

Example = http://www.example.com/foo#fragment
  • it only exists in the browser and is not sent to the server.

Port: the # after the colon. It follows the colon

Example = http://localhost:8000/
  • default port is 80

Headers format = Name: value

Example = GET /foo?p=1 HTTP/1.1
  • Host: www.example.com
  • User - Agent: chrome v17

HTTP Responses

Example = HTTP/1.1 200 ok
  • HTTP/1.1 = status line
  • List of codes
    • 200 = status code
    • 302 = Found
    • 404 = Not Found
    • 500 = Server Error
  • ok = reason phrase
  • HTTP Response Header Examples
    • Date: Tue Mar 2012 04:33:33 GMT
    • Server: Apache /2.2.3
    • Conent-Type: text/html;
    • Content-Length:1539

Servers

  • purpose: respond to HTTP requests
  • response types: static and dynamic
    • static = pre-written file, images (most pages in the early days of the web)
    • dynamic = made on the fly (most pages are like this today) by web application
